# SSH-android KOReader Plugin

SSH server for KOReader on Android, modelled after [simplesshd](https://github.com/pelya/simplesshd). Provides SSH access, `scp` file transfer, and `rsync` sync without root, foreground service, or Android boot receivers.

## Features

- **Bootstrap auth**: on first connect a single-use 8-character password is generated and shown in the KOReader menu. No pre-configuration required.
- **Key-only auth**: once you drop an `authorized_keys` file in the config directory, password auth is automatically disabled on the next server start.
- **scp and rsync**: remote `scp` and `rsync` commands are intercepted and routed to bundled helper binaries.
- **Start with KOReader**: optional autostart setting.
- **Android 4.4+ compatible**: primary target is Tolino (ARMv7, Android 4.4); also works on modern arm64 Android.

## How it works

### Auth flow (mimics simplesshd exactly)

The patched dropbear replaces the standard auth-type selection in `svr-auth.c`:

1. On each new SSH connection, dropbear checks whether `authorized_keys` exists in the config directory **and** is at least 10 bytes long.
2. **No keys file** → password auth mode: 8 cryptographically random characters are generated via `genrandom()` using a 64-char alphabet that avoids visually ambiguous characters (`I l 1 O 0`). The password is stored in the session's auth state and **logged to stderr** (captured to `dropbear.err`). The plugin reads this file and surfaces the password in the menu.
3. **Keys file present** → public-key-only auth: the password code path is never reached.

The password is generated inside dropbear's auth code because that is the only way to inject a runtime password without root on Android — there is no writable `/etc/shadow` and no PAM.

### scp / rsync dispatch

The patched dropbear (`dbutil.c`) intercepts remote commands before exec:

- `scp <args>` → `sh -c "$SSHD_LIBDIR/libscp.so <args>"`
- `rsync <args>` → `sh -c "$SSHD_LIBDIR/librsync.so <args>"`

`$SSHD_LIBDIR` is set by dropbear in the child environment before exec'ing the user shell. The plugin initialises it from the `SSHD_LIBDIR` environment variable (set to `nativeLibraryDir` before launching the binary).

### Android 10+ W^X policy

Android 10 forbids executing files from any app-writable path. Binaries must live in `nativeLibraryDir`, which the package installer places on an exec-capable mount. They are shipped as `.so` files inside `jniLibs/` in the APK:

| On-disk name in APK | Role |
|---|---|
| `libdropbear.so` | SSH server |
| `libscp.so` | scp helper |
| `librsync.so` | rsync helper |
| `libsftp-server.so` | sftp-server helper (optional) |

The plugin locates them at runtime via `android.nativeLibraryDir`.

### File layout at runtime

All runtime state lives under KOReader's data directory:

```
<datadir>/settings/SSH-android/
  authorized_keys          ← put your public key here
  dropbear_rsa_host_key    ← generated on first connection, persistent
  dropbear_ecdsa_host_key
  dropbear_ed25519_host_key
  dropbear.pid             ← PID of running server
  dropbear.err             ← stderr log (bootstrap password lives here)
  dropbear.err.old         ← log from previous session
```

## User guide

### First login (no authorized_keys)

The one-time password is **not** pre-generated at server startup. It is generated by dropbear's auth code the moment a client attempts to connect (`svr_authinitialise()` runs per-connection). This is by design: Android has no `/etc/shadow` and no root PAM, so the only place to inject a runtime password is inside the auth flow itself.

1. Enable the server: **Tools → SSH server (Android)**.
2. Try to connect from your computer (the connection will ask for a password):
   ```
   ssh -p 2222 root@<device-ip>
   ```
3. The connection will prompt for a password. Leave it open or cancel — dropbear has now generated and logged the password.
4. In KOReader, tap **Show one-time password** in the SSH submenu. Enter that password at the SSH prompt.

### Adding your public key

After the first login (or via another file transfer method), place your public key in the config directory:

```
# From your computer, after logging in with the one-time password:
scp -P 2222 ~/.ssh/id_ed25519.pub root@<device-ip>:<datadir>/settings/SSH-android/authorized_keys
```

On the next server start (stop and start again), the auth status item in the menu will show **Auth: public key only** and the password code path is permanently disabled until the file is removed.

### Recommended client config (`~/.ssh/config`)

```
Host koreader
    Port 2222
```

Then: `ssh koreader`, `scp koreader:/path/to/book.epub .`, `rsync -r koreader:/sdcard/books/ ./books/`.

### File transfer

```bash
# scp (single file)
scp -P 2222 book.epub root@<device-ip>:/sdcard/Books/

# rsync (directory sync)
rsync -avz -e "ssh -p 2222" ./books/ root@<device-ip>:/sdcard/Books/
```

## Building the native binaries

The plugin requires a patched dropbear built as a standalone executable, plus scp, rsync, and sftp-server helpers. **Source: the [simplesshd](https://github.com/pelya/simplesshd) repository.**

> The simplesshd repo's dropbear is a JNI shared library (`libsimplesshd-jni.so`), not a standalone binary. We need a standalone executable with a custom entry point (`standalone.c`) that replaces the JNI glue.

### Prerequisites

- Android NDK (tested with the version bundled in `~/Android/Sdk/ndk-bundle` or equivalent)
- The simplesshd repository (contains patched dropbear, rsync, openssh sftp-server sources)

### Step 1 — Write `standalone.c`

Create `jni/standalone.c` in the simplesshd repo. This file replaces `interface.c` for the standalone binary. It declares the globals the patched dropbear references, reads configuration from environment variables, sets up the log file, and calls `dropbear_main()`.

```c
#include <stdlib.h>
#include <string.h>
#include <stdio.h>
#include <unistd.h>
#include <fcntl.h>

/* Globals referenced throughout the patched dropbear source.
 * In simplesshd these are set by the JNI interface; here we read
 * them from environment variables set by the Lua plugin before exec. */
const char *conf_path        = "";
const char *conf_shell       = "/system/bin/sh";
const char *conf_home        = "/data/local";
const char *conf_env         = "";
const char *conf_lib         = "";
int         conf_rsyncbuffer  = 0;
/* Runtime-state dir (pid + log). Falls back to conf_path if unset. */
static const char *conf_run_path = "";

const char *conf_path_file(const char *fn) {
    char *ret = malloc(strlen(conf_path) + strlen(fn) + 2);
    if (ret) sprintf(ret, "%s/%s", conf_path, fn);
    return ret;
}

static const char *run_path_file(const char *fn) {
    const char *base = *conf_run_path ? conf_run_path : conf_path;
    char *ret = malloc(strlen(base) + strlen(fn) + 2);
    if (ret) sprintf(ret, "%s/%s", base, fn);
    return ret;
}

extern int dropbear_main(int argc, char **argv);

int main(int argc, char **argv) {
    const char *e;
    e = getenv("SSHD_CONF_PATH"); if (e && *e) conf_path     = e;
    e = getenv("SSHD_RUN_PATH");  if (e && *e) conf_run_path = e;
    e = getenv("SSHD_LIBDIR");    if (e && *e) conf_lib      = e;
    e = getenv("SSHD_HOME");      if (e && *e) conf_home     = e;

    if (*conf_path) {
        const char *base = *conf_run_path ? conf_run_path : conf_path;
        char log[1024], log_old[1024];
        snprintf(log,     sizeof log,     "%s/dropbear.err",     base);
        snprintf(log_old, sizeof log_old, "%s/dropbear.err.old", base);
        rename(log, log_old);
        unlink(log);
        int fd = open(log, O_CREAT | O_WRONLY, 0600);
        if (fd >= 0) { dup2(fd, STDERR_FILENO); close(fd); }
        /* Close file descriptors Android leaves open. */
        for (int i = 3; i < 255; i++) close(i);
    }

    return dropbear_main(argc, argv);
}
```

### Step 2 — Add the standalone dropbear module to `jni/Android.mk`

Append after the existing `buffersu` module:

```makefile
# Standalone dropbear server for SSH-android KOReader plugin
include $(CLEAR_VARS)
LOCAL_CFLAGS    := -O -DDROPBEAR_VERSION=\"2020.81-koreader\"
LOCAL_MODULE    := dropbear
LOCAL_SRC_FILES := standalone.c $(DROPBEAR_SRCS)
LOCAL_C_INCLUDES:= dropbear dropbear/libtomcrypt/src/headers dropbear/libtommath
LOCAL_LDLIBS    := -lz
include $(BUILD_EXECUTABLE)
```

The `DROPBEAR_SRCS` variable is already defined higher in `Android.mk` — it lists all ~500 dropbear source files including libtomcrypt and libtommath.

### Step 3 — Build

```bash
cd ~/projects/simplesshd

# Generate the default_options guard header (required by the patches)
dropbear/ifndef_wrapper.sh < dropbear/default_options.h > dropbear/default_options_guard.h

# Build all modules for all ABIs
~/Android/Sdk/ndk-bundle/ndk-build -j$(nproc)
```

Outputs land in `libs/<ABI>/`: `dropbear`, `scp`, `rsync`, `sftp-server`, `buffersu`.

### Step 4 — Rename and place for KOReader build

Binaries must be renamed to `.so` to satisfy Android's W^X policy (the installer places `.so` files in `nativeLibraryDir`, which is exec-capable; plain executables in writable paths are not).

```bash
ABI=arm64-v8a      # arm64 for modern Android; armeabi-v7a for Tolino/Android 4.4
LIBS=~/projects/simplesshd/libs/$ABI
KOREADER_INSTALL=~/projects/koreader/install/koreader   # adjust to your build dir

cp $LIBS/dropbear    $KOREADER_INSTALL/dropbear
cp $LIBS/scp         $KOREADER_INSTALL/scp
cp $LIBS/rsync       $KOREADER_INSTALL/rsync
cp $LIBS/sftp-server $KOREADER_INSTALL/sftp-server
```

Then build the KOReader APK as normal (`make android`). The `make/android.mk` `update` target picks these up and renames them to `lib*.so` in `jniLibs/` automatically:

```makefile
# From make/android.mk:
DROPBEAR_BIN  ?= $(INSTALL_DIR)/koreader/dropbear   # → libdropbear.so
SCP_BIN       ?= $(INSTALL_DIR)/koreader/scp         # → libscp.so
RSYNC_BIN     ?= $(INSTALL_DIR)/koreader/rsync       # → librsync.so
```

Each copy step is conditional (`if [ -x ... ]`), so the APK builds cleanly even without the SSH binaries present.

### ABI targets

| Device | ABI | Android version |
|---|---|---|
| Tolino (primary target) | `armeabi-v7a` | 4.4 (API 19) |
| Modern Android phones/tablets | `arm64-v8a` | 5.0+ |
| x86 emulators / Chromebooks | `x86` / `x86_64` | optional |

**Architecture must match the KOReader process ABI.** A 32-bit binary executed from a 64-bit process inherits the arm64 seccomp filter; the first arm32 syscall causes `SIGSYS` and instant silent death. Ship the ABI that matches the KOReader APK being built.

## What the simplesshd patches change

The patched dropbear source differs from upstream in these files:

| File | What changed |
|---|---|
| `svr-auth.c` | Auth type selection replaced: checks `authorized_keys`, either enables pubkey-only or generates and logs a single-use password |
| `svr-authpubkey.c` | Adds `authkeys_exists()`: reads `$conf_path/authorized_keys`, returns true iff file exists and is ≥ 10 bytes |
| `dbutil.c` | `run_shell_command()` intercepts `scp`/`rsync` remote commands and routes them to `libscp.so`/`librsync.so` in `$conf_lib`; `install_environment()` sets `SSHD_LIBDIR` in the child env before exec |
| `localoptions.h` | Host key paths use `conf_path_file()` macro; `DROPBEAR_PIDFILE` is `conf_path_file("dropbear.pid")`; Android PATH; no X11/agent forwarding; `DROPBEAR_DELAY_HOSTKEY` enabled |
| `sshpty.c` | Uses `rename()` instead of `link()` for Android filesystem compatibility |

## Troubleshooting

**Server starts but I cannot connect**
- Check the port is correct (default 2222).
- Verify the device IP: the start notification shows it.
- On Android 10+, confirm `libdropbear.so` is present in the APK (see build steps).

**"Password not found in log"**
- The server may not have started correctly. Check `<datadir>/settings/SSH-android/dropbear.err`.
- If the file is empty or missing, the binary either failed to exec (wrong ABI, missing `.so`) or `SSHD_CONF_PATH` was not set.

**Architecture mismatch / server dies instantly with no output**
- The binary ABI does not match the KOReader process ABI.
- Rebuild `standalone.c` for the correct ABI (see ABI targets above).
- Diagnose: `file libs/arm64-v8a/dropbear` should show `ELF 64-bit LSB executable, ARM aarch64`.

**scp / rsync hangs or fails**
- Confirm `libscp.so` / `librsync.so` are present alongside `libdropbear.so` in `nativeLibraryDir`.
- `SSHD_LIBDIR` must point to the same directory; the plugin sets this automatically on Android.

**Host key changed warning on every restart**
- Host keys are stored persistently in `<datadir>/settings/SSH-android/`. If this directory is wiped, new keys are generated. Add a `HashKnownHosts no` / `StrictHostKeyChecking no` entry for the device in your `~/.ssh/config` if you wipe frequently during development.
